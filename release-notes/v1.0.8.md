## What's Changed

The DeltaForge MCP server: the Model Context Protocol bridge that exposes DeltaForge SQL validation, formatting, lineage, documentation search, and catalog browsing to AI assistants.

### Aligned to engine 1.0.8

- The server version tracks the DeltaForge engine release. The embedded documentation and SQL knowledge are rebuilt from the 1.0.8 corpus, so `search_docs`, `get_command_syntax`, and `validate_syntax` reflect the current command surface.
- Documentation coverage picks up the 1.0.8 engine additions: table usage auditing (`SHOW TABLE USAGE`), the captured table join graph (`SHOW TABLE JOINS`), the `SEMANTIC CONTEXT` command surface, and `SHOW STATS ACTUAL` now reporting disk-spill activity.

### Semantic Context grounding

- New read-only tools let assistants ground natural-language queries in the curated "Semantic Context" spaces: `list_semantic_contexts` enumerates the available contexts, and `get_semantic_context` returns a context's pinned tables, business instructions, virtual key relationships, named expressions, and example queries.
- These tools are additive grounding only; they do not widen access. RBAC stays the access boundary, so an assistant sees only what the user is already permitted to query.
